    Elixir 学习笔记 01 - The differences and commons between Javascript and Elixir

    Wenjing Liu
    2023-05-30 (Updated: 2023-05-30)
    myblog/source/_posts
    Elixir, 学习笔记

    Difference

    Falsy

    • nil, false → Elixir
    • false, 0, -0, 0n, "", null, undefined, NaN → JS

    Default Arguments

    Strict And Non-Strict Operators

    • and, or, not → Only use with booleans
    • &&, || → Non-Strict

    Immutable Data

    • instead of mutating the data, we rebind the variable to new data
      • ?? where is the new data

    Else-If

    Atoms

    • Atoms are named constants. In other words, their name is their value.
    • :atoms
    • Why?
      • One primary reason is performance. Atoms are stored in an atom table and can be referenced by a key. This makes it way faster to check if two atoms are equa.
    • nil, false, true are atoms

    Tuples

    • Tuples are intended as fixed-size containers for multiple elements. To manipulate a collection of elements, use a list instead. Enum functions do not work on tuples.
    • it’s common in Elixir to use tuples to represent failure or success cases with the associated error message or success value.

    Equal =

    • In Elixir, the = sign is the match operator → Pattern Matching
    • In JS, = is the assignment operator .
    • Use an _ to ignore any variables.

    List Operator

    • Merge List
      • [1,2,3] ++ [4,5,6] #Elixir
      • [1,2,3].concat([4,5,6]) //JS
    • Subtract matching elements from a list
      • [1,2,3] -- [2,3] #Elixir
      • [1,2,3].filter(el => ![2,3].includes(el)) // JS
    • List operations evaluate from right to left.
    • Prepending
      • [1 | [2,3]] #Elixir
      • [2,3].unshift(1) //JS
      • Cannot append a number using similar syntax[[1,2] | 3] -> NOT AVAIABLE
    • Get the first elements / last elements
      • [head | _tail] = [1,2,3] -> head is [1] / tail is [2,3]
      • `let list = [1,2,3]; head = list[0];

    Keyword Lists

    • a key-value data structure guarantee order. They are performant for small amounts of data, but less performant for large amounts of data than maps.
    • Keys in a keyword list do not have to be unique.
    • Accessing a key -> list[:atom] -> [name: "Wiki"][:name] // "Wiki"

    Maps and Objects

    • Keys in a map must be unique. Also unlike keyword lists, maps also do not guarantee key order.
    • map = %{:name / 1 / [1,2,3] / "abc" => "Wenjing"}
    • let obj = {name: "Wenjing"}
    • Updating maps - initial = %{name: "wj", num: 2}
      • You can only update existing atom key values in a map, otherwise it will cause an error.
      • %{initial | name: "wm"} #Elixir
      • initial.name = "wm" //JS
    • Collections · Elixir School
    • We can only pattern match on the value in the map, not the key. Keys in maps must be literals such as atoms, strings, tuples, etc.

    Pin Operator

    • To use the literal value of the variable, we can use the pin operator ^.
    • name = "WJ" => ^name // "WJ"

    Underscore

    • The underscore _ is used as a placeholder for a value that is not going to be used. It is often used in pattern matching to ignore certain parts of a value.

    Return a value

    • In Elixir the output of a function is always the last line while in Javascript the output needs a syntax return ...

    Call a anonymous function

    • erlang - Why are there two kinds of functions in Elixir? - Stack Overflow
      • every time you see a def, you get an empty variable scope.
      • Functions in Elixir are identified by name and arity.
    • sum = fn (a, b) -> a + b end => sum.(1, 2)
    • const sum = (a, b) => a + b => sum(1,2)

    Pipe Operator

    • |> -> The pipe operator allows you to take the output of one function and pass it in as an argument for the input of another function
    • If a function is called with multiple arguments, the function piped in will be the first argument.

    Function Arity

    • The number of parameters your function accepts is called the arity of the function.

    Private Functions

    • defp -> 缩写（Define Function Private)

    Callback Functions

    • Capture operator & creates an anonymous function ↗
    • It can’t distinguish between you wanting to reference the function vs calling it. In Elixir, parentheses are optional, so Callback.callback_function is equivalent to Callback.callback_function().

    Scope (need to be investigated)

    • Variables defined in one scope cannot be accessed in another scope.
    • In Javascript, the same variable can be assess inside a function.

    Module Attributes

    • @ - Reads and writes attributes in the current module.

    Doctests

    Struct

    • Structs are extensions built on top of maps that enforce constraints on your data.
    • defstruct ... ->
    • @enforce_keys -> to enforce that certain keys are set.

    Enumeration

    • Enum.map/2 apply our function to each item and produce a new collection
    • Enum.filter/2 to filter the collection to include only those elements that evaluate to true using the provided function
    • Enum.reduce/3 can distill our collection down into a single value

    Converting Non-Enumerables Into Enumerables

    • Integers to list of digits
      Integer.digits(numbers) - Integer.undigits([])

    Comprehensions

    • A comprehension is syntax sugar. That means that while it may look cleaner, it does not provide any additional behavior. The Enum module and comprehensions can accomplish the same behavior.
    • By default, a comprehension collects the result into a list.
    • Enum.map/2, Enum.reduce/2, and Enum.filter/2 can accomplish all of the same functionality as a comprehension. In general, the Enum is more broadly used than comprehensions.

    Reduce

    • Anytime you have a collection, and want to produce a non-list output, Enum.reduce/3 might be the right tool for the job.

    Same

    • Capital letters are always less than lowercase letters regardless of alphabetical order. → "Z" < "a" //true
    • Accessing Map Values - map = %{key: "value"}
      • map.key => Dot notation only works for maps with atom keys
      • map[key] => Bracket notation is especially useful for maps with non-atom keys.
