    Difference

    Falsy

    • nil, false → Elixir
    • false, 0, -0, 0n, "", null, undefined, NaN → JS

    Default Parameter

    Strict And Non-Strict Operators

    • and, or, not → Only use with booleans
    • &&, || → Non-Strict

    Immutable Data

    • instead of mutating the data, we rebind the variable to new data
      • ?? where is the new data

    Else-If

    Atoms

    • Atoms are named constants. In other words, their name is their value.
    • :atoms
    • Why?
      • One primary reason is performance. Atoms are stored in an atom table and can be referenced by a key. This makes it way faster to check if two atoms are equa.
    • nil, false, true are atoms

    Tuples

    • Tuples are intended as fixed-size containers for multiple elements. To manipulate a collection of elements, use a list instead. Enum functions do not work on tuples.
    • it’s common in Elixir to use tuples to represent failure or success cases with the associated error message or success value.

    Equal =

    • In Elixir, the = sign is the match operator → Pattern Matching
    • In JS, = is the assignment operator .
    • Use an _ to ignore any variables.

    List Operator

    • Merge List
      • [1,2,3] ++ [4,5,6] #Elixir
      • [1,2,3].concat([4,5,6]) //JS
    • Subtract matching elements from a list
      • [1,2,3] -- [2,3] #Elixir
      • [1,2,3].filter(el => ![2,3].includes(el)) // JS
    • List operations evaluate from right to left.
    • Prepending
      • [1 | [2,3]] #Elixir
      • [2,3].unshift(1) //JS
      • Cannot append a number using similar syntax[[1,2] | 3] -> NOT AVAIABLE
    • Get the first elements / last elements
      • [head | _tail] = [1,2,3] -> head is [1] / tail is [2,3]
      • `let list = [1,2,3]; head = list[0];

    Keyword Lists

    • a key-value data structure guarantee order. They are performant for small amounts of data, but less performant for large amounts of data than maps.
    • Keys in a keyword list do not have to be unique.
    • Accessing a key -> list[:atom] -> [name: "Wiki"][:name] // "Wiki"

    Maps and Objects

    • Keys in a map must be unique. Also unlike keyword lists, maps also do not guarantee key order.
    • map = %{:name => "Wenjing"}
    • let obj = {name: "Wenjing"}
      • Collections · Elixir School

    Underscore

    • The underscore _ is used as a placeholder for a value that is not going to be used. It is often used in pattern matching to ignore certain parts of a value.

    Same

    • Capital letters are always less than lowercase letters regardless of alphabetical order. → "Z" < "a" //true
